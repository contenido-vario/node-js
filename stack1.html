1. HTML + CSS (TailwindCSS v4) + JS Vanilla + Vite

Tailwind CSS es un framework de CSS de "clases de utilidad" (utility-first). En lugar de ofrecer componentes prefabricados (como botones ya diseñados), proporciona clases de bajo nivel que se aplican directamente en el HTML. Es el estándar de diseño actual.

En el pasado hemos ido utilizando Tailwind css enlazado desde un cdn. Ahora lo instalaremos directamente en el proyecto para aprovechar todo su potencial.

Lo instalaremos haciendo uso de Vite, un servidor de desarrollo y empaquetador (bundler) que agiliza la creación de aplicaciones web. Instalamos Vite y su plugin oficial para Tailwind v4:

pnpm add -D vite @tailwindcss/vite

¿Cómo trabajar con Tailwind v4?

En Tailwind v4, toda la configuración se realiza directamente mediante CSS nativo. Para ello creamos un archivo input.css en la carpeta ./src/ e importamos el framework en la primera línea. Debajo de ella, podemos extender nuestro diseño usando las nuevas directivas:

Dentro de éste archivo css, y debajo de la línea @import "tailwindcss";, agregamos las nuevas directivas de Tailwind v4.

  • @theme: Se usa para crear o extender variables del sistema de diseño (colores, fuentes, animaciones).
  • @layer base: Para modificar los estilos globales de etiquetas HTML nativas (body, h1, etc.)
  • @utility: Reemplaza al antiguo @layer components para crear clases personalizadas y reutilizables.

Desplegar el proyecto en GitHub (método manual tradicional)

Paso 1: Compilar tu código en local (Generar el CSS final)

Antes de subir nada, necesitamos que Node.js y Tailwind generen el archivo CSS real que el navegador puede leer.

  • Abre la terminal en tu editor (Visual Studio Code) y asegúrate de que tu servidor de desarrollo esté apagado (puedes presionar Ctrl + C para apagarlo)

  • Ejecuta el comando de producción:

    pnpm build

  • Verás que en tu proyecto se crea (o se actualiza) una carpeta llamada dist/ que contiene un archivo llamado output.css. ¡Ese es el CSS optimizado y minificado de Tailwind v4!

Paso 2: Crear el repositorio en GitHub

Crea un nuevo repositorio y configuralo de la siguiente manera. Una vez listo, haz clic en el botón Create Repository:

  • Repository name: dale un nombre corto al repositorio en minúsculas preferiblemente
  • Public / Private: Selecciónalo como Public (Obligatorio para que GitHub Pages sea gratis).
  • Opciones iniciales: Deja todo lo demás desmarcado (sin README, sin .gitignore).

Paso 3: Subir los archivos manualmente

Busca un enlace que dice: "uploading an existing file" (subir un archivo existente). Se abrirá una zona donde puedes arrastrar tus archivos.

⚠️ Vamos a arrastrar solo lo que el navegador necesita y lo que configura nuestro proyecto.

NUNCA arrastres la carpeta node_modules (pesa demasiado y romperá la subida).

Arrastra hacia la ventana de GitHub los siguientes archivos y carpetas:

  • Tu archivo index.html (debe quedar en la raíz).
  • Tu archivo package.json.
  • Tu carpeta dist (que creamos en el Paso 1, y que contiene el CSS final).
  • Tus archivos de JavaScript si tuvieras (script.js, etc.).

Paso 4: Encender GitHub Pages

Recuerda hacerlo como siempre: Settings > Pages > Deploy from a branch y seleccionar main (o master) donde dice none.

Deja la carpeta en / (root) y haz clic en Save (Guardar).
