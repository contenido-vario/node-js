Instalar Node.js

Los siguientes seis pasos nos permitirán instalar Node.js utilizando Fast Node Manager (fnm). fnm es un gestor de versiones que permite gestionar múltiples versiones de Node.js de forma rápida, ligera y multiplataforma.

A diferencia de la instalación manual desde la web oficial, fnm evita conflictos entre proyectos y permite cambiar de versión al instante sin necesidad de desinstalar o reinstalar el software.

Además de instalar fnm, usaremos Cmder como terminal.

Usar Cmder implica disponer de un emulador de consola (terminal) portátil y gratuito para Windows que mejora significativamente la experiencia de la línea de comandos mediante una interfaz moderna y personalizable.

Los pasos para la instalación de Node.js (instalar Cmder, fnm, pnpm y la configuración de la terminal de Visual Studio Code) se realizan una sola vez ya que son pasos de instalación de herramientas en el ordenador.

Paso 1. Desinstalar el Node.js existente

- Desinstalar el Node.js viejo

  1. Ve al Panel de Control de Windows > Desinstalar un programa
    (o ve a Configuración > Aplicaciones > Aplicaciones instaladas).
  2. Busca Node.js en la lista.
  3. Haz clic derecho y selecciona Desinstalar.

- Borrar carpetas fantasma

  1. Presiona las teclas Windows R, escribe %appdata% y presiona Enter.
  2. Si ves una carpeta llamada npm o npm-cache, bórralas sin miedo.
Paso 2. Instalar cmder

La página oficial de Cmder es: https://cmder.app/.

Cmder sirve para mejorar drásticamente la experiencia de usar la línea de comandos en Windows. Su función principal es envolver el símbolo del sistema estándar (cmd.exe), PowerShell o Bash, proporcionando una interfaz moderna y funcional sin necesidad de instalar nada en el sistema operativo.

Es ideal para desarrolladores y administradores de sistemas que necesitan un entorno de terminal consistente que puedan llevar consigo en una memoria USB o usar en máquinas donde no tienen permisos de administrador para instalar software.

Cmder incorpora Clink, que añade las funcionalidades de Bash (el intérprete de línea de comandos predeterminado en la mayoría de las distribuciones de GNU/Linux y en macOS) a Windows. Esto significa que se pueden usar indistintamente instrucciones de Bash o de PowerShell, para realizar las mismas operaciones.

La opción más completa y recomendada para reemplazar Cmder en Mac es iTerm2. Es el estándar de la industria para desarrolladores en macOS y ofrece funcionalidades que Cmder intenta emular en Windows.

Cmder no requiere instalación, es portable.

  1. Ve a la página oficial de cmder y descarga la versión Full (que viene con Git integrado).
  2. Descomprime el archivo .zip en la carpeta que desees y ejecuta Cmder.exe.

Agregar Cmder al PATH de Windows

Si quieres poder abrir Cmder desde cualquier lugar de tu computadora e integrarlo bien con tu sistema, sigue estos pasos:

  1. Presiona Win R, escribe sysdm.cpl y pulsa Enter.
  2. Ve a la pestaña Opciones avanzadas.
  3. Haz clic en el botón Variables de entorno... (abajo a la derecha).
  4. En la sección Variables de usuario, busca la que se llama Path y haz doble clic.
  5. Haz clic en Nuevo y pega la ruta de la carpeta donde has guardado Cmder (por ejemplo: C:\Cmder).
  6. Dale a Aceptar a todo.

Ahora podrás ejecutar Cmder desde cualquier carpeta de Windows.

Paso 3. Instalar fnm. Fast Node Manager

fnm o Fast Node Manager, es un gestor de versiones de Node.js moderno, rápido y ligero. Permite instalar y cambiar entre diferentes versiones de Node.js según los archivos .nvmrc o .node-version del proyecto

  1. Abre Cmder, escribe la siguiente instrucción y presiona Enter.
    winget install Schniz.fnm

  2. Si se te pide aceptar unos términos de Microsoft, escribe Y (de Yes) y presiona nuevamente Enter.
    Cierra Cmder.

  3. Tras hacer el winget install, se recomienda explícitamente cerrar todas las terminales y si fnm no se reconoce, es porque winget requiere reiniciar la sesión de usuario o el explorador de Windows, para actualizar las variables de entorno del sistema.

  4. Una vez reiniciada la sesión de usuario, abre nuevamente Cmder para abrir pegar la línea del entorno de fnm en su archivo de configuración user_profile.cmd.
    Abrimos el archivo de configuración en el bloc de notas con esta instrucción:
    notepad %CMDER_ROOT%\config\user_profile.cmd

  5. Ve hasta el final de ese archivo de texto que se abrió, pega esta línea en una nueva línea después del @echo off:
    @fnm env --use-on-cd --shell cmd | call

  6. Guarda el archivo (Ctrl G o selecciona Archivo > Guardar), cierra el bloc de notas y cierra Cmder.

En macOS/Linux fnm se instala de forma nativa con un solo comando de curl (disponible en su web oficial), y puedes saltar el paso de Cmder

Paso 4. Instalar la última versión estable de Node.js

Cierra Cmder (si no lo has cerrado) por última vez y vuélvelo a abrir para integrar completamente fnm. Ya podemos instalar la versión más estable de Node.js escribiendo la siguiente instrucción en Cmder:

fnm use --install-if-missing lts-latest

Paso 5. Instalar pnpm

Ejecuta este comando en tu Cmder:

npm install -g pnpm

Cuando termine de instalar escribe: pnpm -v para confirmar que la instalación ha sido correcta. Si te devuelve su número de versión, ya tenemos vía libre para comenzar a programar en Node.js

Tu entorno de desarrollo está listo:

  • Cmder como terminal
  • fnm gestionando las versiones de Node.js
  • pnpm como el motor para gestionar los paquetes (aplicaciones) que uses en tu proyecto.
Paso 6. Configuración de la terminal de Cmder en VS Code

Visual Studio Code necesita que configuremos cmder explícitamente en su propia lista de perfiles para que aparezca en el menú desplegable de la terminal y poderlo seleccionar. Para ello:

  1. Presiona Ctrl , (Ctrl coma) en VSCode para abrir la Configuración.

  2. Busca el icono pequeño que parece un archivo con una flecha girando (ver title: Open Settings JSON) y clica sobre el mismo. Se abrirá un archivo de texto lleno de configuraciones.

  3. Al final de todo, añade una coma , después de la última propiedad existente y, justo antes de la última llave de cierre }, pega el siguiente bloque de código.

    ⚠️ ¡Atención a la ruta!. Si guardaste Cmder en una carpeta diferente a C:\Cmder, debes cambiar la ruta en la línea de los args. Las dobles barras diagonales \\ son obligatorias en los archivos JSON para evitar errores de escape.

  4. Guardamos el archivo con Ctrl S y abrimos la terminal de Visual Studio Code Ctrl Ñ

  5. Por defecto, Visual Studio Code suele abrir la terminal clásica de Windows (PowerShell o cmd). Para usar Cmder configurado con fnm dentro de Visual Studio Code, hacemos clic en la flechita que está al lado del botón + en la esquina de la terminal (abajo a la derecha), y seleccionamos Cmder.

Cambiar el tamaño de la letra de la terminal

Para cambiar únicamente el tamaño de la letra de la terminal, sigue estos pasos:

  1. Volvemos a presionar Ctrl , en VS Code para abrir la Configuración.

  2. En la barra de búsqueda de arriba, escribimos: Terminal Font Size.

  3. Se mostrará una opción que dice Terminal > Integrated: Font Size
    que por defecto viene en 14.

  4. Escribe el tamaño de texto que te sea cómodo (se actualiza al instante).
